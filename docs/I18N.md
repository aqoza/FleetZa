# Internationalization & RTL

FleetManage ships English + Arabic with full right-to-left support. The system is
**type-safe** and **module-scoped** so every new feature is translatable and
direction-correct by default. This doc is the contract for adding strings, modules,
and languages.

## How it works

- `src/i18n/index.tsx` — the engine. Merges every namespace into `en` / `ar`
  dictionaries, derives `MessageKey` (the union of all English keys), and exposes
  the `I18nProvider`, `useI18n()` and `useT()` hooks.
- `src/i18n/messages/{en,ar}/<namespace>.ts` — the dictionaries. One namespace per
  module. Each `ar/<ns>.ts` is typed `Record<keyof typeof en<Ns>, string>`, so a
  missing Arabic translation is a **compile error**.
- Direction: Arabic sets `<html dir="rtl">`. Layout follows via Tailwind **logical
  utilities** — never hardcode a physical side.
- Language persists to `localStorage` (instant, pre-login) and to
  `profiles.language` for signed-in users (synced by `AuthContext`).
- Formatting (money, dates, numbers) stays driven by the tenant **country locale**
  (`shared/countries.ts`) and keeps Western/Latin digits for business clarity —
  independent of UI language.

## Using strings in a component

```tsx
import { useT } from "@/i18n"; // or relative path
function MyThing() {
  const t = useT();
  return <button>{t("vehicles.add")}</button>;
}
```

Interpolation: `t("reports.addedN", { count: 3 })` where the value is
`"Added {count} renewals"`.

Enum labels come from `src/lib/labels.ts` as translation keys:
```tsx
<Badge tone={vehicleStatus[v.status].tone}>{t(vehicleStatus[v.status].labelKey)}</Badge>
```

## Adding a new module (future features)

1. Create `src/i18n/messages/en/<module>.ts`:
   ```ts
   export const en<Module> = { "<module>.title": "Title", ... } as const;
   ```
2. Create `src/i18n/messages/ar/<module>.ts`:
   ```ts
   import { en<Module> } from "../en/<module>";
   export const ar<Module>: Record<keyof typeof en<Module>, string> = { "<module>.title": "…" };
   ```
3. Register both in `src/i18n/index.tsx` (one import + one spread in `en` and `ar`).
4. Prefix keys with the module name to avoid collisions (`vehicles.*`, `fuel.*`).

That's it — `t()` autocompletes the new keys, and TS forces the Arabic side complete.

## Adding a new language

Add the code to `Language` and `LANGUAGES` in `src/i18n/index.tsx`, create
`src/i18n/messages/<lang>/*.ts` mirroring `en`, register them, and set `dir` if the
script is RTL. The migration `check (language in (...))` on `profiles.language` must
be widened too.

## RTL rules (apply in every component)

Use **logical** Tailwind utilities so layout mirrors automatically. Never use the
physical variants below on the left column:

| Physical (avoid) | Logical (use) |
|---|---|
| `pl-`, `pr-` | `ps-`, `pe-` |
| `ml-`, `mr-` | `ms-`, `me-` |
| `left-`, `right-` | `start-`, `end-` |
| `text-left`, `text-right` | `text-start`, `text-end` |
| `rounded-l-`, `rounded-r-` | `rounded-s-`, `rounded-e-` |
| `border-l`, `border-r` | `border-s`, `border-e` |

- **Direction-bearing icons** (back arrows, chevrons, "next/prev") must flip:
  add `rtl:-scale-x-100` to the icon (or swap `ChevronLeft`/`ChevronRight`).
- **Charts** (recharts) read left-to-right even in Arabic — wrap chart containers
  in `<div dir="ltr">` so axes/legends stay sane.
- `gap-`, `flex`, `grid`, `justify-between` are already direction-agnostic — leave them.
- `flex-row` reverses automatically under `dir="rtl"`; rely on that instead of
  reordering DOM.
